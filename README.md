# Auth Dashboard Template

Minimal TanStack Start template with:

- Better Auth
- Convex
- protected dashboard routes
- shadcn/ui components and utilities

## Recommended Use

Use this repo as a GitHub template repository.

That gives each new app:

- a fresh repository with no shared history
- the same starter structure
- no need to maintain a separate scaffolding package

## Create A New Project

### Option 1: GitHub Template

1. Mark this repository as a GitHub template in repository settings.
2. Click `Use this template` in GitHub.
3. Create a new repository from it.
4. Clone the new repository locally.

### Option 2: Bun CLI From GitHub

You can scaffold from it with Bun:

```bash
bun create JMRBDev/auth-dashboard-template my-new-app
```

## Post-Create Checklist

After creating a new project from this template, update these immediately:

1. Rename the package in `package.json`.
2. Replace the app name in `src/routes/__root.tsx`, `src/routes/index.tsx`, and `src/components/app-sidebar.tsx`.
3. Replace the Google OAuth provider if your project needs different auth providers.
4. Update `.env.example` for the new deployment URLs and auth credentials.
5. Replace this README with project-specific documentation.

## Scripts

```bash
npm run dev
npm run build
npm run lint
npm run typecheck
npm run check
npm run convex:dev
npm run convex:codegen
```

`npm run check` is the baseline verification command.

## Environment

Current env reads in the codebase:

- `CONVEX_DEPLOYMENT`
- `VITE_CONVEX_URL`
- `VITE_CONVEX_SITE_URL`
- `SITE_URL`
- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`

## Generated Files

- `convex/_generated/*` is generated by Convex.
- `src/routeTree.gen.ts` is generated by TanStack Start.

Do not edit generated files by hand.

## Local Setup

1. Install dependencies with `bun install` or `npm install`.
2. Copy `.env.example` to a local env file and fill in the required values.
3. Start Convex with `npm run convex:dev`.
4. Start the app with `npm run dev`.
